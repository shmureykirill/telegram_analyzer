СБОР ДАННЫХ

Данные берутся только с публичных страниц https://t.me/s/<username> — без входа в аккаунт и без MTProto.

Пагинация истории реализована через параметр ?before=<post_id>, что позволяет листать архив канала до 4 000+ сообщений.

Между запросами выдерживается случайная пауза 1–3 с, обрабатывается HTTP 429.

АНАЛИЗ ТЕКСТА (NLP)
  1. Очистка: удаление URL, @упоминаний, #хэштегов, пунктуации, цифр
  2. Токенизация: split по пробелам после очистки
  3. Стоп-слова: русские + английские (NLTK) + базовые белорусские
  4. Ключевые слова: топ-50 унаграмм по частоте
  5. N-граммы: биграммы и триграммы с топ-20/10
  6. Классификация тем: словарный метод (topics.json) — если ≥2 ключевых слова темы встречаются в тексте, тема назначается с весом = кол-ву совпадений

Словарь тем редактируется в config/topics.json без изменения кода.

ПРОГНОЗ ТРЕНДОВ

Алгоритм сравнивает текущий интервал (6 ч) со скользящим средним за предыдущие 4 интервала:

СигналПорогВывод
Рост упоминаний>150%📈 Рост
Рост ср. просмотров>200%📈 Рост
Активность инфлюенсеров≥1 канала с ср.просм ≥5 000📈 Рост
Рост реакций на пост>50%Усиливающий сигнал
Падение упоминаний>50%📉 Спад

Горизонт прогноза — ближайшие 24 часа. Все прогнозы сохраняются в БД с временной меткой для истории.

ОГРАНИЧЕНИЯ ПРОТОТИПА
  • Только публичные каналы — приватные группы недоступны
  • Профили интересов — на уровне каналов, не отдельных пользователей
  • Репосты недоступны в публичном HTML (заглушка = 0)
  • Тематика определяется словарём, ML-модели не используются
  • Прогноз на основе простых порогов, не статистических моделей
